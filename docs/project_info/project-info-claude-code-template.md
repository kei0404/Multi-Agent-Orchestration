## プロジェクトの目的・背景 (Purpose/Context)

社内開発チームで Claude Code を導入する際のテンプレートとして使用する。

### ユーザーが得られる価値
- Claude Code の設定を効率的に行える
- 社内統一のルールに基づいた開発が可能になる
- チーム間で一貫した Claude Code の運用ができる

## コア機能仕様 (Core Functionality Specifications)

### カスタムコマンド一覧

| コマンド | 説明 |
|----------|------|
| `interview-claude` | CLAUDE.md 作成のための情報収集インタビュー |
| `generate-plans` | PLANS.md（実行計画書）の生成・更新 |
| `create-claudeagent-for-engineer` | エンジニア向け CLAUDE.md 生成 |
| `create-claudeagent-for-business` | ビジネス向け CLAUDE.md 生成（エンジニアリング固有設定を省略した簡素版） |
| `e2e-test` | E2Eテスト実行 |

## 前提条件・ルール (Top-Level Rules)

### 開発環境
- 案件ごとに決定される（汎用テンプレートとして設計）

### テスト・検証ルール
- コード変更時は必ずテストを実行する
- テストコードも一緒に作成する

### 指示・承認ルール
- PLANS.md に記載のない作業は実行前に確認を求める
- 大規模な変更（複数ファイルにまたがる変更）は事前承認が必要
- 曖昧な指示には必ず確認を取る
- 特定の操作（削除、リファクタリング等）は明示的な許可が必要

## 禁止事項 (Constraints/DON'TS)

### Git リポジトリ管理
- `git push --force` の禁止
- `main` / `master` ブランチへの直接コミット禁止
- 未レビューのコードのプッシュ禁止

### 機密情報の取り扱い
- `.env` ファイルのコミット禁止
- APIキーやパスワードのハードコーディング禁止
- 認証情報を含むファイルの読み取り/出力禁止

## プロンプト設計指針 (Prompt Design Guidelines)

特に規定なし（案件ごとに決定）
