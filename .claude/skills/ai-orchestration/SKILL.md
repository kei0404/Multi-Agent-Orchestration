# AI Orchestration

複数のAIエージェントを協調させてプロジェクトを効率的に進行するスキル。

## コア機能

### 1. マルチエージェント協調

複数のAIエージェントを目的に応じて使い分ける：

| エージェント | 得意分野 | 使用場面 |
|-------------|---------|---------|
| Claude | 分析・設計・コード生成 | メイン開発、アーキテクチャ設計 |
| Codex | コードレビュー・最適化 | セカンドオピニオン、品質チェック |
| Gemini | 調査・分析・レビュー | 外部視点、代替案検討 |

### 2. コンテキスト管理

エージェント間でコンテキストを共有・同期する：

```
共有コンテキスト:
├── CLAUDE.md      - プロジェクトルール
├── specs/         - 仕様書
├── plans/         - 実行計画
└── docs/          - ドキュメント
```

### 3. ワークフローパターン

#### パターン A: シーケンシャル実行

```
Claude (設計) → Codex (レビュー) → Claude (修正) → Gemini (最終確認)
```

#### パターン B: 並列実行

```
       ┌→ Codex (パフォーマンス分析)
Claude ┼→ Gemini (セキュリティ分析)
       └→ Claude (機能実装)
           ↓
       結果統合
```

#### パターン C: 専門家会議

```
問題提起 → 各エージェントが意見 → 最適解の統合
```

## 実行手順

### Step 1: タスク分析

1. タスクの複雑さを評価
2. 必要なエージェントを特定
3. 実行パターンを決定

### Step 2: コンテキスト準備

1. 共有コンテキストを読み込む
2. エージェント固有の指示を準備
3. 出力フォーマットを統一

### Step 3: エージェント実行

各エージェントに対して：
- 明確な役割と責任を指定
- 期待する出力形式を指定
- 「確認不要、具体的な提案まで出力」を末尾に追加

### Step 4: 結果統合

1. 各エージェントの出力を収集
2. 矛盾点を特定・解決
3. 最終的な成果物を生成

## エージェント呼び出し例

### Codexでコードレビュー

```bash
codex exec --full-auto --sandbox read-only --cd /path/to/project \
  "以下のコードをレビューしてください。[コード内容]
   確認や質問は不要です。具体的な改善案とコード例まで自主的に出力してください。"
```

### Geminiで設計レビュー

```bash
gemini-cli "以下の設計についてレビューしてください。[設計内容]
確認や質問は不要です。具体的な改善提案まで自主的に出力してください。"
```

## ベストプラクティス

1. **単一責任**: 各エージェントに明確な役割を与える
2. **コンテキスト最小化**: 必要な情報のみを渡す
3. **出力標準化**: 統一フォーマットで結果を収集
4. **フィードバックループ**: 結果を次のエージェントに引き継ぐ
5. **エラーハンドリング**: 失敗時のフォールバック戦略
