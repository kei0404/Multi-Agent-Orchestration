以下の手順で要件定義書を作成してください:

1. 現在のディレクトリ構造を確認
2. 既存のドキュメントを参照
3. $ARGUMENTS の内容を要件として取り込む
4. spec.md テンプレートに従って要件定義書を作成
5. docs/spec.md として保存

## 条件
- spec-writingを使用して作成する
- `.claude/docs/templates/spec.md` のテンプレートにしたがって作成する
- 作成した要件定義は、`docs/specs`内に`spec.md` として保存する
- 既に要件定義が存在する場合は、`spec_**.md`として保存する
    　記入例：spec_02.md

## 注意:
- 曖昧な表現を避ける
- 機能は入力・処理・出力で記述
- 完了条件を明確にする

##  要件定義のアンチパターン

❌ 避けるべき記述
```
## 機能
- 使いやすいUI
- 高速なレスポンス
- セキュアな認証
```

✅ 推奨される記述
```
## 機能
### 1. ユーザー検索機能
- 入力: 氏名（部分一致）、部署（完全一致）、スキル（複数選択）
- 処理: 条件に合致する社員を検索し、最大50件返却
- 出力: 社員ID、氏名、部署、メールアドレス
- 制約: 検索結果は500ms以内に返却すること
```
